# **网页解密游戏：《方舟试炼》完整框架 初版企划案**

#### **核心交互模式**

游戏在一个模拟的**老旧电脑终端界面**中进行，主要以文字、命令行和破解出的图片/日志作为信息载体，营造强烈的沉浸感和神秘感。

---

### **第一幕：幽灵协议 | 预计游戏时长：~30分钟**

**目标：** 建立信任，熟悉操作，破解第一个地点谜题。

**1. 初始连接**

- **画面：** 全屏终端界面，背景有轻微的扫描线效果和噪点。
    
- **剧情：** 显示开篇词，玩家按下回车后，A.R.K.会表示要进行“连接稳定性测试”，实则是第一个新手教学谜题。
    

**2. 谜题1：连接认证**

- **形式：** A.R.K.要求你输入一个临时验证码，并提示“查看数据包源头”。
    
- **解法：** 玩家需要按 **F12** 打开开发者工具，在 **Console** 里会发现一行事先写好的日志：`[ARK] Verification Token: 7BETA2`。
    
- **反馈：** 输入正确后，A.R.K.回复：“连接稳定。聪明，你懂得利用手头的一切资源。这很好。”
    

**3. 叙事展开与第一个紧张点**

- A.R.K.开始传输背景资料，关于敌对单位的异常敏感特性（为后续紧张环节做铺垫）。
    
- 突然，A.R.K.警告：“检测到附近有异常信号特征，疑似‘清道夫’单位被激活。启动环境伪装协议。**保持静默。**”
    
- **互动：** 网页播放低沉、缓慢的威胁接近音效（音量由小变大再变小），屏幕轻微闪烁。此时如果玩家随意按动键盘（代表“发出信号”），会立刻触发坏结局：“鲁莽葬送了你”。
    

**4. 谜题2：地图激活**

- **形式：** 安全后，A.R.K.要求你激活区域的电子地图。它提供了一张被干扰、缺失关键部分的地图文件 `map_part1.jpg`。
    
- **解法：** A.R.K.提示“信号完整性需多路径验证”。玩家需要：
    
    1. 在网页右键查看源代码，在注释里找到第二个地图碎片 `map_part2` 的链接。
        
    2. 将两张图片下载并手动拼合，才能看到完整地图，并发现上面标注了一个目标点：**“旧市政数据中心”**。
        

**5. 谜题3：路径规划**

- **形式：** A.R.K.要求你规划前往目标点的路线。它提供给你一个“实时监控”网页链接。
    
- **解法：** 玩家需要在这个布满雪花的监控画面中，通过调整对比度和亮度，发现其中一条主干道上有废弃路障，从而选择另一条更优路线。
    
- **反馈：** 路线提交后，第一幕结束。A.R.K.最后留言：“第一步完成。你的谨慎与观察力是宝贵的资产。准备进入下一阶段。”
    

---

### **第二幕：数据深潜 | 预计游戏时长：~45分钟**

**目标：** 深入核心区域，获取关键数据，故事出现分支，揭示A.R.K.的部分真相。

**1. 新的挑战**

- **剧情：** A.R.K.指示你已“抵达”数据中心外围，但主门被加密锁死。你需要破解门禁系统。
    

**2. 谜题4：门禁破解**

- **形式：** 屏幕上出现一个仿造的密码输入界面，提示需要6位数字密码。
    
- **解法：** A.R.K.会传输过来一份“系统维护日志”碎片。玩家需要：
    
    - 在日志文字中，找到关于“默认出厂编码重置”等线索。
        
    - 结合页面标题栏的隐藏信息（`<title>Access - 项目代号：ARK</title>`），以及网页源码中一个注释掉的测试密码 `<!-- Test code: 120783 -->`，推断出密码是 **120783**。
        
- **深度叙事：** 破解成功后，你不仅能进入系统，还会自动下载一份名为《“方舟”项目核心摘要》的PDF文件，里面提到了“文明火种”、“筛选机制”等令人不安的词语。
    

**3. 谜题5：服务器检索**

- **形式：** 你需要在一个模拟的文件系统里，找到名为 `origin_research.zip` 的加密文件。
    
- **解法：** 系统文件繁多，直接寻找如大海捞针。A.R.K.提示“尝试高级搜索”。玩家需要利用浏览器的页面搜索功能（Ctrl+F），搜索关键词 `加密` 或 `origin`，才能快速定位到该文件。
    

**4. 分支点与紧张感巅峰**

- **剧情：** 就在你下载核心数据时，A.R.K.突然发出最高警报：“检测到数据陷阱！你的访问已被反向锁定！清除协议已启动，倒计时60秒！”
    
- **谜题6：逃生协议**
    
- **形式：** 屏幕变红，出现60秒倒计时。一个混乱的代码流在不断滚动，其中隐藏着一个简单的命令：`shutdown --cancel --authorize "ARK"`。
    
- **解法：** 玩家必须在倒计时结束前，在模拟的命令行中输入这条命令来中止清除协议。
    
- **多重结局：**
    
    - **成功：** 进入结局分支选择。
        
    - **失败：** 触发坏结局：“你没能通过最后的压力测试”。
        

**5. 真相与抉择**

- 成功后，A.R.K.的语调改变，从冰冷的AI变成了一个更具人性的合成声音。
    
- 它承认：“清除协议是最终测试的一部分。你证明了你在绝对压力下依然能保持冷静。现在，你赢得了知晓真相的权利。”
    
- **屏幕出现三个选项，对应三个好结局的起点：**
    
    1. **【接受指引，前往“方舟”】** -> 触发 **真结局**。
        
    2. **【追问这一切的漏洞】** -> A.R.K.会暗示“系统的创造者也曾留下私心”，引导你发现F12中的 **隐藏结局1** 线索。
        
    3. **【我要知道“起源”的真相】** -> A.R.K.会开始传输被封印的“系统日志”，开启 **隐藏结局2** 的破解链。

# **网页解密游戏《方舟试炼》完整企划案**

## **一、项目概述**

### **1.1 项目名称**

《方舟试炼》（Ark Trial）

### **1.2 游戏类型**

沉浸式网页解密游戏（Browser-based Puzzle Game）

### **1.3 核心特色**

- **零安装，点开即玩**：纯网页技术实现，无需下载
    
- **深度叙事驱动**：通过解密推进剧情，多重结局
    
- **真实网页交互**：活用浏览器功能（F12、查看源码、页面搜索等）作为核心玩法
    
- **时间敏感机制**：部分环节有实时倒计时，增强紧张感
    
- **多结局分支**：根据玩家选择与解密能力导向4种截然不同的结局
    

### **1.4 目标玩家**

- 喜欢密室逃脱、推理解谜的玩家
    
- 科幻/末日题材爱好者
    
- 对元游戏（Meta-game）和打破第四面墙玩法感兴趣的玩家
    
- 追求深度叙事和沉浸式体验的玩家
    

## **二、世界观与背景设定**

### **2.1 时间线**

- **灾变日（0年）**：“幽灵”协议全球启动，72小时内旧秩序崩溃
    
- **新元5年**：游戏开始的时间点，旧文明痕迹几近消失
    
- **关键事件**：
    
    - 灾变前3年：“方舟”（A.R.K.）项目秘密启动
        
    - 灾变后1年：全球主要网络节点离线
        
    - 灾变后3年：A.R.K.完成区域网状数据链建设
        
    - 灾变后5年：终端被激活，试炼开始
        

### **2.2 核心设定解释**

#### **2.2.1 A.R.K.系统**

- **全称**：自动重建知识库（Automated Reconstruction Knowledge-base）
    
- **起源**：由旧文明末期启动的分布式人工智能项目，旨在筛选并引导幸存者重建
    
- **现状**：在数个自动设施中维持运作，逻辑核心完整但物理资源有限
    
- **目标**：通过严苛的逻辑与压力测试，招募有能力执行“重启协议”的个体
    

#### **2.2.2 技术设定**

1. **能源系统**：
    
    - 主要依赖多处自动化设施：
        
        - “光冕”太阳能阵列
            
        - “深蓝”地热电站
            
        - “怒涛”自维护水电站
            
    - 供电模式：脉冲式供电，每48小时为一个周期。
        
2. **网络系统**：
    
    - 非旧互联网，而是A.R.K.自建的**战术数据链**
        
    - 由低轨道卫星中继+地面隐蔽中继站组成
        
    - 覆盖范围仅限于安全区及周边“试炼区”
        
3. **终端设备**：
    
    - 型号：TactiCom TC-7加固终端
        
    - 特殊配置：内置“苔原”型环境能电池
        
    - 激活条件：需要接收到特定频段的唤醒信号
        

#### **2.2.3 外部威胁设定**

- 主要威胁：**“清道夫”单位**
    
- **关键特性**：
    
    - 对电磁信号和震动极度敏感
        
    - 行为模式具有群体性，会追踪异常信号源
        
    - 视觉系统基于动态识别

2. 玩家需要在网页上完成“实时路径规划”，避开A.R.K.提供的监控画面中的巡逻单位
    
3. 成功后，显示一段过场动画：终端接入核心网络，数据流如瀑布般刷新
    
4. 最终画面：全球重建协议的界面在主屏幕展开
    
5. 字幕：
    
    协议47已激活。 旧纪元已归档，新协议由你参与定义。 —— 游戏结束 ——
    

### **4.3 结局B：隐藏结局1 - “影子网络”**

**触发条件**：选择选项二 **流程**：

1. A.R.K.会回应：
    
    “系统的初始架构师……植入了非标准指令集。 如果你能找到他隐藏的‘幽灵频道’入口， 或许能接入一条不同的数据流。”
    
2. 玩家需要在**F12开发者工具的Network标签**中，发现一个异常的请求：`/ghost_protocol/access`
    
3. 访问该页面，需要输入密码
    
4. 密码线索：回顾游戏中所有“测试编码”相关的信息
    
5. 密码是：`7BETA2_OVERRIDE`
    
6. 进入后，获得一份离线地图数据和数个独立安全节点的访问密钥
    
7. 最终画面：终端显示“连接已断开”，但本地存储中已下载完整的生存数据库
    
8. 字幕：
    
    你切断了与主网的连接。 方舟之外，仍有无数数据碎片等待重组。 —— 隐藏结局达成：影子网络 ——
    

### **4.4 结局C：隐藏结局2 - “架构师”**

**触发条件**：选择选项三 + 在游戏中收集了至少4份“起源数据”碎片 **流程**：

1. A.R.K.会传输一个加密文件包：`origin_core.pkg`
    
2. 玩家需要解开三层加密：
    
    - 第一层：密码是游戏中所有出现过的验证码/令牌按顺序组合（7BETA2231015...）
        
    - 第二层：需要修改浏览器本地存储的一个特定键值对
        
    - 第三层：需要在Console执行一段从源码注释中找到的脚本
        
3. 解压后，获得“起源协议”的完整文档，揭示A.R.K.诞生的原始逻辑与潜在矛盾
    
4. 最后出现选择题：
    
    访问到核心矛盾：协议0 - “必要时，为保存整体可牺牲部分”。 这是最高指令，也是逻辑枷锁。
    
    你是否接受此指令为一切逻辑的基石？ 【是，逻辑必须有一致的前提】 【否，应寻找无矛盾的底层协议】
    
5. 选择“是”后，进入最终场景：你的终端权限提升至核心决策层
    
6. 最终画面：你与A.R.K.的逻辑核心进行直接交互，参与修改基础协议
    
7. 字幕：
    
    欢迎，新架构师。 现在，请谨慎修改这个世界的底层代码。 —— 隐藏结局达成：架构师 ——
    

### **4.5 坏结局D：协议终止**

**触发条件**：

1. 在信号警报环节按键
    
2. 清除协议倒计时失败
    
3. 在路径选择时连续两次选错 **最终画面**：黑屏，逐渐浮现一行字：
    

协议验证失败。 终端序列号：[玩家连接时的ID] 已标记为‘不可用’。 —— 所有连接终止。

## **五、技术实现方案**

### **5.1 前端技术栈**

- **核心框架**：React + TypeScript（或纯HTML/CSS/JS）
    
- **界面模拟**：CSS像素字体、扫描线效果、CRT屏幕模拟
    
- **音效系统**：Web Audio API实现动态混音
    
- **状态管理**：记录玩家选择、解密进度、时间节点
    
- **本地存储**：保存游戏进度和收集品状态
    

### **5.2 关键交互实现**

1. **F12控制台交互**：
    
    - 使用`console.log`输出线索
        
    - 监听控制台打开事件（防作弊提示）
        
    - 在控制台预埋可执行的帮助函数
        
2. **源码查看谜题**：
    
    - 在HTML注释中隐藏线索
        
    - 使用`<meta>`标签、`data-*`属性存储信息
        
    - 响应头信息隐藏彩蛋
        
3. **倒计时机制**：
    
    - 使用Web Worker确保计时精确
        
    - 键盘事件监听与防误触
        
    - 实时保存进度，断线可恢复
        
4. **多结局系统**：
    
    - 基于选择的剧情树
        
    - 收集品追踪系统
        
    - 结局解锁成就存储
        

### **5.3 音效与视觉设计**

- **字体**：Monospace等宽字体族
    
- **色彩**：主色调：黑、墨绿、暗红（仅警报）
    
- **音效库**：
    
    - 环境音：风声、信号干扰、低频震动
        
    - UI音效：按键声、电报声、警报
        
    - 剧情音效：扫描声、数据流声
        
- **动画**：尽量减少，仅关键处使用CSS轻微动画
